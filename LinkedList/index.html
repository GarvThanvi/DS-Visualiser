DoublyLinkedList BinarySearch
Aim: Implement a Singly Linked List.
Theory: Like arrays, Linked List is a linear data structure. Unlike arrays, linked list elements are not stored at a contiguous location; the elements are linked using pointers.
Arrays can be used to store linear data of similar types, but arrays have the following limitations.
  1. The size of the arrays is fixed: So we must know the upper limit on the number of elements in advance. Also, generally, the allocated memory is equal to the upper limit irrespective of the usage.
  2. Inserting a new element in an array of elements is expensive because the room has to be created for the new elements and to create room existing elements have to be shifted.

Algorithms
  • Insertions at start
    • Create a new node with the given value.
    • Set the next pointer of the new node to point to the current head of the list.
    • Update the head pointer to point to the new node.
  • Insertions at End
    • Create a new node with the given value.
    • If the list is empty (head is None), set the head pointer to the new node.
    • Otherwise, traverse the list from the head until you reach the last node.
    • Set the next pointer of the last node to point to the new node.
  • Insertions in Between
    • Create a new node with the given value.
    • If the position is 0, set the next pointer of the new node to point to the current head of the list and update the head pointer to the new node.
    • Otherwise, traverse the list from the head while keeping a count of the nodes visited.
    • Stop when you reach the node just before the desired position (position - 1).
    • Set the next pointer of the new node to point to the node at position 'position'.
    • Update the next pointer of the node at position 'position - 1' to point to the new node.
  • Deletion at End
    • If the list is empty (head is None), print an error message.
    • If there is only one node in the list (head.next is None), set the head pointer to None.
    • Otherwise, traverse the list from the head until you reach the second-to-last node.
    • Set the next pointer of the second-to-last node to None.
  • Deletion from Between
    • If the list is empty (head is None), print an error message.
    • If the position is 0, update the head pointer to point to the second node (head.next).
    • Otherwise, traverse the list from the head while keeping a count of the nodes visited.
    • Stop when you reach the node just before the node to be deleted (position - 1).
    • Update the next pointer of the node at position position - 1 to point to the node at position 'position + 1'.
